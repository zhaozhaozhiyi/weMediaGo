题目：媒体资源处理工具箱开发
一、项目背景和目标

1.1 项目背景

在融合媒体快速发展的当下，内容生产场景日益多元化，自媒体、教育、电商等行业对轻量化、高效化的媒体资源处理工具需求激增。传统媒体处理工具多为客户端重型软件，存在操作门槛高、适配性差等问题，难以满足移动端、Web端即时处理的场景需求。

本次AI编程竞赛要求前端工程师在4小时内，借助AI编程助手开发一款Web端媒体资源处理工具箱，核心目标是结合AI辅助开发优势，快速落地贴合融合媒体业务场景的轻量化处理功能，兼顾开发效率与产品实用性。

1.2 项目目标

•业务目标：交付轻量级 Web 工具，支持图片/视频基础编辑+导出，满足自媒体、教育等场景的“快速修图+即时分享”核心诉求，操作门槛 ≤3 步。
•技术目标：借助AI编程助手（如Copilot、通义千问等）快速生成核心逻辑代码，实现可运行Web应用（含核心功能）+ AI辅助证据（代码生成日志）+ 3步操作验证视频；4小时内完成核心功能开发及基础UI搭建。

二、功能要求

本工具箱以“基础必备+场景扩展”为原则设计功能，核心功能需确保完整可运行，扩展功能可根据开发时间灵活落地，具体要求如下：

2.1 核心功能模块（必实现）

2.1.1 图片处理工具集（适配H5）
1.1 裁剪功能

•比例裁剪
￮内置高频比例：3:4（竖版短视频）、21:9（电影屏）、4:5（小红书封面）、1:2（长条图）、9:16（抖音封面），1:1（正方形）、4:3（常规）、16:9（宽屏）总计8种比例，支持一键切换，切换后裁剪框实时适配图片尺寸。
￮操作优化：比例切换时自动居中裁剪框，鼠标悬浮比例选项时显示对应场景提示（如“4:5-小红书封面”）。
•自由裁剪
￮交互能力：裁剪框支持鼠标拖拽边缘/角点缩放，拖拽框内区域平移；
￮辅助功能：裁剪框内实时显示当前选中区域的宽高像素值；
•裁剪历史
￮记录最近5次裁剪操作，支持撤销&重做。

1.2 旋转与翻转

•固定角度旋转
￮保留原功能：0°/90°/180°/270°一键旋转，新增“旋转重置”按钮，一键恢复图片初始角度。
￮扩展能力：新增角度输入框，支持手动输入0-360°任意整数角度，输入后点击“应用”实时生效，输入非法值（如负数、小数）时弹窗提示。
•细粒度旋转
￮新增旋转滑块：范围0-360°，拖动过程中图片实时跟随旋转，滑块旁显示当前角度值；支持鼠标点击滑块任意位置快速跳转角度。
•翻转对比
￮支持水平/垂直翻转；支持一键还原操作。

1.3 缩放与尺寸调整

•比例缩放
￮控制方式：提供0.1-5.0倍缩放滑块（步长0.1），同步显示缩放倍数输入框（支持手动输入0.1-5.0的小数），滑块与输入框联动（修改其一，另一自动同步）。
•鼠标交互缩放
￮新增鼠标滚轮控制：滚轮向上滚动时图片放大（每次滚动递增0.1倍），向下滚动时缩小（每次递减0.1倍）；
￮边界限制：缩放后图片最小尺寸不小于100×100px，最大尺寸不超过原图尺寸的5倍，超出限制时弹窗提示。

二、特效处理模块

2.1 马赛克功能

•选区类型
￮保留原类型：矩形、圆形选区，新增选区切换按钮（图标化展示，点击切换）。
￮新增选区：自由手绘选区（鼠标长按拖拽绘制任意形状，松开后闭合选区）、多边形选区（点击图片生成顶点，双击最后一个顶点闭合选区，支持最多20个顶点）。
•操作优化
￮选区编辑：选中已创建的选区后，支持拖拽移动位置、拖拽边缘/顶点调整大小；右键点击选区弹出“删除”“隐藏”选项，隐藏后选区变为半透明虚线，便于查看图片背景。
￮参数调节：除模糊度滑块（0-100）外，新增“马赛克颗粒大小”滑块（1-20像素，值越大颗粒越粗），两个参数独立调节，实时预览效果。
2.2 滤镜效果

•基础滤镜库
￮保留原滤镜：复古、清新、黑白、高饱和，新增8种常用滤镜：胶片（模拟胶卷质感）、日系（低饱和、高亮度）、冷色调（偏蓝绿）、暖色调（偏红黄）、怀旧（泛黄复古）、柔光（降低对比度，增加亮度）、锐化（增强边缘细节）、去雾（提升清晰度，减少朦胧感），总计13种滤镜，以图标+文字形式展示。
￮滤镜预览：鼠标悬浮滤镜图标时，图片实时预览该滤镜效果（预览不影响当前编辑状态），点击后应用效果；
•滤镜参数调节
￮强度控制：每个滤镜新增“强度滑块”（0-100%，100%为默认效果），调节后实时更新效果；如“锐化”滤镜强度越高，边缘越清晰。
￮混合效果：支持同时叠加2种滤镜（如“复古+柔光”），叠加后显示“已叠加[滤镜1]+[滤镜2]”，可点击“移除”按钮取消其中一种滤镜。
•自定义滤镜
￮预设保存：调节好滤镜强度或混合效果后，点击“保存预设”，输入预设名称（如“小红书风格”），可保存最多5组自定义预设；预设列表以卡片形式展示，支持重命名、删除。
￮快速应用：点击预设卡片即可一键应用保存的滤镜效果，适合高频场景复用。

2.3 文字水印

•基础样式设置
￮文字输入：支持中英文、数字及特殊字符输入，输入框实时显示水印预览；支持最大输入长度10字符，超出时提示“文字过长，请精简”。
￮颜色与透明度：颜色选择支持色板点击、十六进制代码输入（如#FF0000）、RGB值输入；透明度滑块（0-100%，0%为完全透明，100%为不透明），实时预览水印透明度效果。
•位置与对齐
￮固定位置：保留原“居中、左上、右上、左下、右下”5个固定位置，点击位置选项后水印实时跳转。
￮边距调节：针对固定位置，新增“边距输入框”（10-50像素），如“右上”位置可设置“距右20px、距上15px”，精确控制水印位置。
•高级样式扩展
￮文字背景：支持添加文字背景色（颜色选择同文字颜色），背景透明度独立调节（0-100%）；
￮文字平铺：设置横线&倾斜，调整平铺间距大小。
2.4 额外特效

•色彩调节
￮亮度调节：滑块范围0-200%（100%为原图亮度），低于100%降低亮度，高于100%提升亮度；调节时实时显示“亮度：XX%”。
￮对比度调节：滑块范围0-200%（100%为原图对比度），低于100%降低对比度（画面更灰），高于100%提升对比度（明暗差异更明显）。
￮饱和度调节：滑块范围0-200%（100%为原图饱和度），0%为黑白效果，高于100%增强色彩鲜艳度。
￮色温调节：滑块范围0-200%（100%为原图色温），低于100%偏冷色（蓝绿色调），高于100%偏暖色（红黄色调）；新增“冷色”“暖色”“默认”三个快捷按钮，一键切换常用色温。

三、图片优化

3.1 压缩优化

•压缩控制
￮滑块调节：保留0-100%压缩率滑块（0%为无压缩，100%为最大压缩），滑块旁实时显示“当前压缩率：XX%”。
￮快捷档次：新增3个压缩档次按钮，与滑块联动：
▪高清晰：10-30%压缩率（适合需保留细节的场景，如产品图）。
▪平衡：30-60%压缩率（兼顾清晰度与体积，如公众号配图）。
▪体积优先：70-100%压缩率（优先减小体积，如缩略图、表情包），确保1MB图片压缩后≤300KB，5MB图片压缩后≤1.5MB。
•实时反馈
￮体积预估：压缩前显示原图大小（如“原图大小：2.4MB”），调节压缩率时实时显示“预估压缩后大小：XXKB/MB”。
￮对比展示：压缩完成后，弹窗显示“压缩前后对比”，包含原图大小、压缩后大小、压缩率（如“压缩率：75%”），支持“重新压缩”或“确认导出”。

3.2 图片优化

•导出设置
￮导出质量：导出时可选择“默认质量”“高质量”“低质量”，确保导出文件符合需求。
￮文件名确认：导出前弹窗显示“确认导出”，包含文件名、格式、预计大小，支持修改文件名后再导出；导出文件自动下载到浏览器默认下载目录，下载完成后显示“下载成功”提示。

2.2 扩展功能模块（可选实现）

•视频互动：支持弹幕添加（自定义文字、颜色、字体、滚动速度，提供顶部固定、底部固定、滚动三种模式）；支持弹幕互动功能——对已有弹幕点击“回复”后，输入内容可作为“子弹幕”跟随原弹幕显示，形成互动链路。
•音频可视化：支持MP3、WAV 格式音频上传，并自动解析时间、采样率，生成可视化预览，支持3种可视化预览（3D粒子动效、2D波形图、频谱图），播放时可视化效果与音频实时同步，暂停保持当前可视化效状态，支持进度条拖拽定位、定位后可视化效果随音频片段更新；

三、技术栈

3.1 基础技术栈

•前端基础：HTML5、CSS3、JavaScript（ES6+语法）。

五、评测标准

本次评测从功能完整性、技术实现质量、用户体验、AI工具应用效率四个维度展开，总分100分，具体评分标准如下：
一、核心评分维度（100分）

1. 功能完整性（30分）

•图片处理工具集（15分）：含基础编辑（裁剪、旋转等）、特效处理（马赛克、滤镜等）、格式与优化（压缩、导出等）。
•扣分规则：每缺失1个核心子功能扣2分，功能异常（如裁剪失真、滤镜失效）扣1-3分。
•扩展功能加分：每实现1个扩展功能（如视频弹幕互动）加5分，最多加10分。

2. 技术实现质量（30分）

•性能优化（12分）：媒体处理无明显卡顿（预览延迟≤0.5秒）得6分；大文件（图片≥5MB、视频≥100MB）处理无崩溃、无内存溢出得6分。
•兼容性（8分）：Chrome、Edge浏览器完美运行（无样式错乱、功能失效）得5分；适配1920/1290/1440分辨率/h5等界面自适应得3分。
•代码质量（10分）：代码结构规范（pages+components+utils+assets）、注释完整得6分；无冗余代码、命名规范得4分。

3. 用户体验（25分）

•操作便捷性（13分）：支持拖拽+点击双重上传得4分；参数调节用直观控件（滑块、下拉框）得5分；导出按钮位置醒目得4分。
•反馈清晰度（12分）：操作过程有加载动画、成功/失败提示得6分；双屏预览、历史记录复用等辅助功能正常得6分。

4. AI工具应用效率（15分）

•AI使用合理性（6分）：开发文档明确AI核心应用场景（如生成裁剪算法、UI组件），且与技术选型契合，每缺失1项关键场景说明扣2分。
•效率提升效果（9分）：复杂逻辑（如弹幕互动）快速落地（≤30分钟）、重复编码减少60%以上，不符合则扣3-9分。

二、扣分项（直接扣减总分）

•核心功能缺失≥2个（未在4小时内完成）：扣30分。
•代码无法本地运行（依赖缺失、配置错误）或核心功能失效（文件损坏、浏览器崩溃）：扣50分。
•未提交开发说明文档：扣10分；文档缺失关键信息（如AI应用记录、核心逻辑），每缺失1项扣3分。
